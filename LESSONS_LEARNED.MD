# Lessons Learned

## Sprint 1

- Every class should have one responsibility.
- Test components individually before integrating them.

## Sprint 2

- Returning objects is more flexible than returning dictionaries.
- Configuration should live outside the code.

## Sprint 3

- The Application class keeps startup logic in one place.
- Git commits should represent meaningful milestones.

## Sprint 4

- Factories reduce coupling between components.
- Good architecture makes future features easier to add.
